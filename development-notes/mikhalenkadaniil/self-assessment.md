В рамках проекта я взял на себя **реализацию компонента AI Chat**, решив использовать новые для меня технологии, чтобы получить максимум практического опыта. Моя работа охватила полный цикл разработки: от проектирования архитектуры и создания схем до написания фронтенда и бэкенда с нуля.

### Инструменты и технологии
В процессе работы я использовал широкий стек современных инструментов:
*   **Frontend:** Фреймворк **Angular** (изучал с нуля в процессе работы), механизмы **Signals** для управления состоянием, **TypeScript** и **WebSockets** для реализации чата в реальном времени.
*   **Backend:** Фреймворк **NestJS** (также осваивал с нуля), **PostgreSQL** для хранения истории сообщений и **OpenRouter API** для интеграции различных LLM-моделей.
*   **Проектирование и документация:** Для описания логики приложения я использовал **Mermaid** (вместо draw.io) для создания человекочитаемых диаграмм user-flow, которые удобно ревьюить на GitHub.

### Что было сделано с нуля
Многие элементы системы я спроектировал и реализовал самостоятельно:
1.  **Интерактивный UI чата:** Создал базовые компоненты формы отправки и разметки сообщений.
2.  **Директива для безопасного парсинга Markdown:** Реализовал логику обработки потокового ответа (streaming) от нейросети, которая корректно отображает текст по мере его генерации.
3.  **Архитектура Store:** Разработал механизм хранения данных на фронтенде, используя **Dependency Injection** Angular вместо NgRx.
4.  **Backend-сервис на WebSockets:** Написал логику общения с LLM через веб-сокеты, чтобы изолировать сложную обработку потоков на сервере и отправлять на клиент только готовые чанки текста.
5.  **Модуль истории сообщений:** Реализовал полноценный бэкенд-модуль с таблицами в БД, репозиториями и SQL-запросами для сохранения и загрузки диалогов пользователя.
6.  **Универсальный провайдер OpenRouter:** Создал прокси-слой, позволяющий пользователям подключать свои API-ключи для доступа к мощным моделям без нагрузки на сервер проекта.

### Сложности в процессе работы
Основным вызовом стало **отсутствие опыта в Angular и NestJS**, что требовало постоянного обучения на ходу. Среди технических трудностей можно выделить:
*   **Специфика Angular:** Например, неожиданное поведение свойства `display: inline` по умолчанию для `:host` или борьба с ошибкой **NG0100** из-за неправильного использования флагов в обход сигналов.
*   **Обработка стриминга:** Было сложно реализовать корректный парсинг Markdown, когда теги могут обрываться в середине потока.
*   **Тестирование:** Написание тестов на Angular, особенно для симуляции `AsyncIterable`, заняло много времени из-за отсутствия предварительного опыта в этой области.
*   **Командные процессы:** Из-за разных часовых поясов, графиков работы и нехватки свободного времени (как у меня, так и у участников команды) возникали задержки в код-ревью. К тому же это первый командный опыт разработки проекта

Несмотря на сжатые сроки и высокую нагрузку, мне удалось реализовать работающий прототип AI Chat с сохранением истории и поддержкой внешних API.
