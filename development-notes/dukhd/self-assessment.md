## 🛠 Work Description & Feature Components
Этот проект стал для меня точкой профессиональной трансформации: от базовых знаний разметки до создания архитектурно сложного **Full-stack приложения** на стеке **Angular** + **NestJS**. В роли **Team Lead** и ведущего разработчика я прошла через все этапы жизненного цикла ПО: от проектирования инфраструктуры монорепозитория и дизайна в Figma до написания серверной логики и тонкой настройки UX-анимаций. Мой фокус был направлен на чистоту кода, масштабируемость системы и инклюзивность интерфейса.

### 1. Инфраструктура и DevOps

- **Инициализация монорепозитория:** Настроила структуру проекта на базе **NPM Workspaces**, объединив фронтенд (Angular), бэкенд (NestJS) и общие типы (shared).
- **Автоматизация контроля качества:**
    - Внедрена система Git-хуков через **Husky**.
    - **Pre-commit:** автоматическая проверка линтинга (ESLint, Prettier) и типов (typecheck).
    - **Pre-push:** реализован кастомный Bash-скрипт для защиты основной ветки (валидация имен веток, запуск тестов во всех воркспейсах).
    - **Commitlint:** строгий контроль формата сообщений коммитов.
- **Документация:** Создала стандарты разработки для команды, включая структуру папок, правила именования веток и шаблон Pull Request.

### 2. Дизайн и UI-Kit

- **Проектирование:** Разработала в Figma полный [дизайн-макет](https://www.figma.com/design/eK5bfZBxMZEyRYodB8GXZl/Final?node-id=0-1&t=Tk0FvdcgD7aP5Tt5-1) лендинга, и др. Подготовила Style Guide (цвета, типографика, переменные). Описала все переменные в соответствующих scss файлах [PR #39](https://github.com/jsgods-rs-tandem/rs-tandem/pull/39#issue-3977829216) [PR #162](https://github.com/jsgods-rs-tandem/rs-tandem/pull/162#issue-4108706856).
- **Интеллектуальный движок компонентов:**
    - Реализовала «умный» полиморфный компонент **Button**, который на основе входных данных автоматически выбирает HTML-тег (`button` или `a`), управляет безопасностью ссылок и состояниями доступности.
    - Разработала набор высокоуровневых компонентов: **Accordion**, **Logo**, **Card**, обеспечивающих консистентность интерфейса и переиспользование кода.
    - Внедрена системная поддержка тем (Light/Dark) через CSS-переменные с синхронизацией с настройками ОС и сохранением в LocalStorage ([PR #73](https://github.com/jsgods-rs-tandem/rs-tandem/pull/73)).

### 3. Frontend-разработка и UX-системы

- **Adaptive Navigation System (Complex Component):** Разработала комплексную систему навигации, включающую десктопный Header и сложное мобильное меню. Система управляет состояниями авторизации и ролями пользователей, динамически изменяя доступные маршруты.
- **Smart Anchor Navigation:** Реализовала **ScrollSpy Service** на базе `IntersectionObserver`. Это обеспечило плавную якорную навигацию с автоматическим отслеживанием и подсвечиванием активных разделов при скролле.
- **Global Layout System (Rich UI Screen):** Интегрировала Header, Sidebar и Footer в единую систему макетов с поддержкой контекстной навигации.
- **Интерактивный Landing Page:** Построила главную страницу на базе разработанного UI-Kit, внедрив множество блоков.
- **Страница 404:** Создала кастомную страницу ошибки с логикой перенаправления и адаптивным дизайном.
- **Система анимаций:**
    - Использовала **Angular Animations** для обработки состояний мобильного меню.
    - Внедрена передовая технология **View Transitions** для бесшовных переходов между страницами. Реализовала программный обход `AbortError`, возникающих при прерывании транзакций (например, при быстрой навигации по якорным ссылкам).
- **Доступность и i18n:** Обеспечила полную поддержку **RU/EN** локализации для всех компонентов, внедрив семантические **aria-labels** и корректное управление фокусом (A11y).

### 4. Backend и архитектура данных

- **SQL-миграции и схема данных:** Самостоятельно расширила схему базы данных PostgreSQL, реализовав миграции для поддержки профилей пользователей, хранения путей к аватарам и интеграции с GitHub-аккаунтами ([PR #168](https://github.com/jsgods-rs-tandem/rs-tandem/pull/168)).
- **Сервисный слой (API Layer):** Разработала `ProfilesService` на NestJS, выделив бизнес-логику обработки данных в отдельный слой, изолированный от контроллеров.
- **Паттерн Facade:** Внедрил паттерн **Facade** на фронтенде для управления состоянием профиля. Это позволило полностью скрыть сложность работы со стором и API от UI-компонентов, сделав их «глупыми» (presentational) и упростив тестирование.
- **Оптимизация производительности:** Реализовала логику кэширования данных на уровне фасада, что исключило избыточные сетевые запросы при повторных переходах в профиль.
